# 个人实训总结报告

## 一、项目概述

本次实训项目开发了一个名为"随机挑战生成器"的社交打卡应用。用户每天接收随机挑战，完成打卡后可分享到广场、与好友互动。技术栈选用了 Next.js 14 + Ant Design 5 作为前端，Python Flask 作为后端 API，Supabase 提供数据库、认证和文件存储服务。项目部署在腾讯云 Windows Server 上，通过计划任务实现开机自启。

## 二、遇到的最大困难及解决过程

整个开发过程中，最具挑战性的问题有两个：

### 1. 好友列表"显示0好友"的诡异 Bug

项目开发后期，用户反馈了一个非常奇怪的问题：好友列表在刚添加好友时能正常显示，但过了一段时间后刷新就显示为0。最让人困惑的是，这个问题不是必现的——有时好有时坏。

经过多次排查，我最终发现了两个叠加的 Bug：

第一个 Bug 在 `backend/routes/friends.py` 中。查询消息记录时，`SELECT` 语句只选了 `content, created_at, sender_id` 三个字段，但后续代码却访问了 `receiver_id`。当用户有消息记录时，Python 会抛出 `KeyError`，导致整个接口返回 500。由于刚添加好友时还没有消息记录，所以不会触发这个错误；一旦用户之间发了消息，接口就崩溃了。这就是"过段时间才显示0"的原因。

第二个 Bug 与 JWT token 过期有关。初始实现中，当用户的 access_token 过期后，`get_current_user()` 直接返回 `None`，导致所有接口返回 401。而前端 `getToken()` 函数优先返回本地缓存的过期 token，没有尝试刷新。修复方案是：后端在 token 过期时仍然从 JWT 中提取 `user_id`（只是不设置 `g.jwt_token`），然后 `get_auth_supabase()` 自动降级为 service_role key 绕过 RLS 限制；前端则改为优先调用 `refreshSession()` 获取新鲜 token。

### 2. 图片存储桶不存在的隐性问题

评论和聊天图片上传功能完成后，用户反馈图片"发不出去"。代码层面没有任何报错，但数据库中 `photo_url` 字段始终为空。经过排查，发现是 Supabase Storage 的 `challenge-photos` 桶从未被创建过。由于前端上传代码使用 `if (!uploadErr) { photoUrl = ... }` 的写法，上传失败时静默跳过，用户完全感知不到错误。这个问题的教训是：前端代码应该有完善的错误反馈机制，不能让失败被静默吞掉。

## 三、对 AI 辅助编程的心得体会

### 效率提升

AI 辅助编程极大缩短了项目开发周期。在传统开发模式下，从零搭建一个包含认证、社交、实时聊天、徽章系统的全栈应用至少需要 2-3 周。借助 AI，我在几天内就完成了主体功能开发。AI 在以下方面表现尤为突出：

1. **代码生成**：Ant Design 组件使用、Supabase API 调用模式、Flask 路由写法等模板化代码，AI 可以一次性生成完整实现
2. **Bug 定位**：当遇到"好友列表显示0"这样的复杂问题时，AI 可以快速分析代码路径，定位到 `receiver_id` 字段缺失这样的根因
3. **全栈衔接**：AI 能够同时理解前后端代码，确保 API 请求格式和响应处理的一致

### 注意事项

AI 辅助编程也有一些局限性需要注意：

1. **需要人工审查安全逻辑**：AI 生成的 JWT 验证代码被设置为 `verify_signature=False`，这是一个严重的资质安全问题，如果没有 Code Review 很难发现
2. **上下文限制**：随着项目变大，AI 可能会忘记之前的架构决策，导致生成不一致的代码
3. **调试能力有限**：对于需要实际运行才能发现的问题（如 Storage 桶不存在），AI 无法提前预判

总体而言，AI 辅助编程更像是有一个"24 小时在线的资深结对编程伙伴"，可以在很大程度提升效率，但最终代码质量和安全责任仍然在开发者身上。

## 四、个人贡献

在本项目中，我完成了以下工作：

1. **全栈架构设计**：设计了 Next.js 14 App Router + Flask + Supabase 的技术架构，规划了 40 个 API 接口和 9 个前端页面
2. **后端开发**：独立完成了 Flask 后端的全部开发工作，包括认证系统、挑战系统、点赞评论、好友聊天、徽章系统等 5 个功能模块
3. **前端开发**：实现了 9 个前端页面的交互逻辑和 UI 设计，包括登录页（含密码强度校验）、Dashboard（含骨架屏和徽章弹窗）、广场 Feed、好友聊天、个人主页等
4. **数据库设计**：设计了 Supabase PostgreSQL 的完整 schema，包括 8 张业务表、RLS 策略、种子数据和复合索引
5. **Bug 修复与优化**：定位并修复了 JWT 过期导致的好友列表消失、图片上传静默失败、评论图片变量引用顺序等 Bug
6. **部署运维**：在腾讯云 Windows Server 上完成部署，配置 Nginx 反向代理和计划任务自启
7. **文档编写**：编写了项目 README、API 接口文档（40 个端点）、AI Prompt 日志等

## 五、总结

通过这次实训项目，我不仅掌握了 Next.js 14 + Flask + Supabase 的全栈开发技能，更重要的是学会了如何利用 AI 工具提升开发效率。我深刻体会到，AI 辅助编程不是简单的"复制粘贴"，而是需要开发者具备设计能力、问题分解能力和代码审查能力，才能发挥 AI 的最大价值。
